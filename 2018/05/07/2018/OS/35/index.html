3.5 产生死锁的原因和必要条件

3.5.1 产生死锁的原因

  1. 资源竞争
  2. 进程间推进顺序不当
  1. 竞争不可抢占性资源引起死锁
  2. 经侦临时性(消耗性)资源引起死锁
  3. 进程推进顺序不当引起死锁
  • 竞争资源,未必产生死锁
  • 是否产生死锁,还取决于动态执行和应用程序细节

3.5.2 产生死锁的必要条件

  1. 互斥条件 : 进程对分配到的资源进行排他性使用
  2. 请求和保持条件
  3. 不剥夺条件
  4. 环路等待条件

3.5.3 处理死锁的基本方法

  1. 预防死锁
    • 设置某些限制条件,破坏四个必要条件中的一个或几个
    • 优点:
      • 容易实现
    • 缺点:
      • 资源利用率低或吞吐量降低
  2. 避免死锁
    • 再资源动态分配过程中使用某种方法防止系统进入不安全状态
    • 优点
      • 较弱限制条件可获得较高系统资源利用率和吞吐量
    • 缺点
      • 有一定实现难度
  3. 检测死锁
    • 预先不采取任何限制,也不检查是否已进入不安全区,通过设置检测机构,检查出死锁后解除
  4. 解除死锁
    • 常用撤销或挂起一些进程,回收一些资源
