1、掌握瀑布、增量、迭代、敏捷等不同软件过程模型的特征和优缺点。

  • 瀑布模型

    • 1
      • 使用广泛,以文档驱动
      • 用来规范软件开发活动
      • 很多模型都是在此基础上改进的
    • 特征
      • 阶段间具有顺序性和依赖性
      • 推迟实现的观点
    • 优点
      • 每个阶段都有与其相关联的里程碑和可交付产品
      • 每个阶段结束前完成文档审查,以及早改正错误
    • 缺点
      • 各个阶段的划分完全固定,阶段之间产生大量的文档,极大增加了工作量
      • 由于开发模型是线性的,用户只有等到整个过程的末期才能见到开发成果,从而增加了开发的风险
      • 无法适应需求不明确和需求的变化
      • 不能反映实际的开发方式,软件开发需要迭代

  • 增量模型

    • 优点
      • 增量概念的引入,不需要提供完整的需求,只要有一个增量出现,开发就可以进行
      • 在项目的初始阶段不需要投入太多的人力资源
      • 产品逐步交付,软件开发能够较好地适应需求的变化
      • 能够看到软件中间产品,提出改进意见,减少返工,降低开发风险
      • 软件能够更早投入市场
      • 开放式体系结构,便于维护
    • 缺点
      • 每个增量必须提供一下系统功能,开发者很难根据客户需求给出大小适合的增量
      • 软件必须具备开放体系结构
      • 易退化成边做边改的方式,使软件过程控制失去整体性
        -

  • 敏捷开发

    • 优点

      • 对变化和不确定性有更快速更敏捷的反应
      • 在快速的同时保持可持续的开发速度
      • 能较好的适应商业竞争环境下对项目提出的有限资源和有限开发时间的约束
    • 缺点
      • 极限编程中的测试驱动开发可能会导致系统通过了测试但不是用户所期望的
      • 重构而不降低系统体系结构的质量是困难的
  • 迭代????

2、软件的功能性需求与非功能性需求分别是什么，非功能性需求一般包含什么内容，需求分析文档一般都包含哪些内容。

  • 功能性需求
    • 描述一个系统的行为,与系统的功能性相关.
  • 非功能性需求
    • 详细描述了一个系统的性能特点.
    • 包含
      • 性能需求:如响应时间,并发量
      • 质量属性:可靠性,可维护性
      • 对外接口:硬件接口,软件接口,数据库接口
      • 约束:硬件设施,编程语言

  • 需求分析文档一般包含哪些内容

    • 引言
      • 文档目的
      • 文档约定
      • 预期的读者和阅读建议
      • 产品范围
      • 参考文献
    • 综合描述
      • 产品背景
      • 产品功能以及优先级
      • 用户特征
      • 运行环境
      • 设计与现实上的限制
      • 假设和依赖性
    • 需求描述
      • 功能需求
      • 数据需求
      • 性能需求
      • 外部接口
      • 设计约束
      • 软件质量属性
      • 其他需求

    • 附录

    • 索引

3、用例图，(功能建模)

  • 要明确用例与参与者的含义，
    • 用例代表功能，(不是单独的事务)
    • 参与者代表系统边界外使用功能的人或系统。
  • 能够区分一个系统中应包含哪些用例，应包含哪些参与者。
  • 要掌握用例之间的include关系和extend关系，箭头不能画错，
    • A include B，箭头从A指向B，B是A的一部分；
    • A extend B，箭头从A指向B，B是基用例，A是扩展用例，A在某些条件下会执行。

4、顺序图（时序图）

  • 用一个二维图描述系统中各个对象之间的交互关系，要明确纵轴线是关联到对象的，
  • 消息用从一个对象的生命线到另一个对象生命线的箭头表示，注意消息有先后顺序。
    要掌握两个要点：
    • 一个是有哪些对象需要交互消息，每一个对象都是一条纵轴线，
    • 另一个是对象间传递的消息名称和顺序，时间上越先执行的消息越在上面。

5、数据流图，(功能建模)

  • 数据流图用图形化方法表示系统功能及功能之间的数据流转关系，
  • 构成要素包括
    • 外部项、
    • 数据加工、
    • 数据存储，
  • 需要明确掌握着三个要素的定义与功能以及三者之间的关系。
    • 数据加工即为系统功能模块，
    • 外部项主要指系统的用户角色，
    • 数据存储是存储数据的模块，比如文件系统或数据库文件。
  • 要掌握分层数据流图之间上层与下层的关系，
    • 一般上层中某个数据加工可以有一个下层数据流图，相对应的上下层数据流图的外部项一般相同。第0层数据流图一般只有一个数据加工，即为整个系统的名称。

6、程序流程图最重要的是清楚各个图形的含义，以及如何配合描述判断、循环等控制逻辑。

7、掌握等价类的划分方法及设计测试用例的方法，

  • 要掌握通过划分等价类来确定测试用例的几个步骤：
    • （1）确定有效等价类；
    • （2）确定无效等价类；
    • （3）对所有等价类列表，并标号；
    • （4）设计测试用例，使其覆盖尽可能多的有效等价类，重复这个过程，直至所有有效等价类被覆盖完；
    • （5）设计测试用例，使每个测试用例均只覆盖一个无效等价类，重复这个过程，直至所有无效等价类被覆盖完。掌握边界值法。

8、掌握不同的软件测试方法，包括单元测试、集成测试、回归测试等。

9、掌握软件项目管理的定义与方法，如何对项目进行WBS分解，编制项目进度网络图，什么是项目的关键路径。

10、软件规模度量中需要掌握KLOC方法、功能点法以及COCOMO方法，清楚不同COCOMO方法的使用场景和计算方法。如何计算工作量，以及如何计算开发时长。

11、CMMI方法是什么意思，五级CMMI分别是什么。

  • 能力成熟度模型
  • 五级
    • 初始级
    • 可重复级
    • 已定义级
    • 量化管理
    • 优化级
