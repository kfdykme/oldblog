3.8 死锁的检测与解除

发表于 2018-05-07

3.8 死锁的检测与解除

  • 如果系统不愿意附加太多约束条件预防死锁,也不希望系统额外开销预测并避免死锁,那么只能允许死锁出现,然后解除它
  • 因此,需要有某种方法来检测死锁

3.8.1 死锁的检测

  1. 资源分配图
  2. 死锁定理 : S为死锁当且仅当S状态的资源分配图是不可完全简化的
  3. 死锁检测中的数据结构
    1. 可利用资源向量Available ,他表示m类资源中每一类资源的可用数目
    2. 把不占用资源的进程(Available = 0)记入L表中,即L_i ∪L
    3. 从进程集合中找到一个Request_i < work的进程,作如下处理
      1. 将其资源分配图简化,释放出资源,增加工作向量Work=Work+ Available
      2. 将他记入L表
    4. 若不能把所有进程都记入L表,便表明系统状态s的资源分配图是不可完全简化的.因此,该系统即将发生死锁

3.8.2 死锁的解除

  • 常用两种解除死锁方法

    1. 剥夺资源,从其他进程剥夺足够数量的资源给死锁进程,以解除死锁状态
    2. 撤销进程,最简单的撤销进程的方法,是使全部死锁进程都夭折掉;或按某种顺序逐渐逐个撤销进程,直至有足够的资源可用,使死锁状态消除为止

  • 按照解除死锁复杂度递增的顺序列出解除死锁的方法

    1. 撤销死锁进程
    2. 把死锁进程恢复到前一个检查点,重新执行每个进程
    3. 按某种原则逐个选择死锁进程进行撤销,直到解除系统死锁
    4. 按照某种原则逐个剥夺进程资源,直到解除死锁

  • 最小代价原则

    • 上述第三种第四种方法需要选择系统付出代价最少的进程,最小代价原则
      • 到目前为止，花费处理机的时间最少的进程；
      • 到目前为止，产生输出最少的进程；
      • 估计未执行部分最多的进程；
      • 到目前为止，已获得资源量最少的进程；
      • 优先级最低的进程。

3.6 预防死锁

  • 预防死锁的方法是使四个必要条件中的第2、3、4条件之一不能成立，来避免发生死锁。至于必要条件1，因为它是由设备的固有属性所决定的，不仅不能改变，还应加以保证。

3.7 避免死锁

发表于 2018-05-07

3.7 避免死锁

3.7.1 安全状态

  • 安全状态，是指系统能按某种进程顺序(P1, P2, …，Pn)(称〈P1, P2, …, Pn〉序列为安全序列)，来为每个进程Pi分配其所需资源，直至满足每个进程对资源的最大需求，使每个进程都可顺利地完成。如果系统无法找到这样一个安全序列，则称系统处于不安全状态。

-

3.7.2 利用银行家算法避免死锁

问题描述：
该算法可用于银行发放一笔贷款前，预测该笔贷款是否会引起银行资金周转问题。
这里，银行的资金就类似于计算机系统的资源，贷款业务类似于计算机的资源分配。银行家算法能预测一笔贷款业务对银行是否是安全的，该算法也能预测一次资源分配对计算机系统是否是安全的。
为实现银行家算法，系统中必须设置若干数据结构。

银行家算法思想——死锁避免策略：
1.当前状态下，某进程申请资源；
2.系统假设将资源分给该进程，满足它的需求；
3.检查分配后的系统状态是否是安全的，如果是安全，就确认本次分配；如果系统是不安全的，就取消本次分配并阻塞该进程。（第三步又称安全算法）
避免死锁策略也称作银行家算法。

3.5 产生死锁的原因和必要条件

发表于 2018-05-07

3.5 产生死锁的原因和必要条件

3.5.1 产生死锁的原因

  1. 资源竞争
  2. 进程间推进顺序不当
  1. 竞争不可抢占性资源引起死锁
  2. 经侦临时性(消耗性)资源引起死锁
  3. 进程推进顺序不当引起死锁
  • 竞争资源,未必产生死锁
  • 是否产生死锁,还取决于动态执行和应用程序细节

3.5.2 产生死锁的必要条件

  1. 互斥条件 : 进程对分配到的资源进行排他性使用
  2. 请求和保持条件
  3. 不剥夺条件
  4. 环路等待条件

3.5.3 处理死锁的基本方法

  1. 预防死锁
    • 设置某些限制条件,破坏四个必要条件中的一个或几个
    • 优点:
      • 容易实现
    • 缺点:
      • 资源利用率低或吞吐量降低
  2. 避免死锁
    • 再资源动态分配过程中使用某种方法防止系统进入不安全状态
    • 优点
      • 较弱限制条件可获得较高系统资源利用率和吞吐量
    • 缺点
      • 有一定实现难度
  3. 检测死锁
    • 预先不采取任何限制,也不检查是否已进入不安全区,通过设置检测机构,检查出死锁后解除
  4. 解除死锁
    • 常用撤销或挂起一些进程,回收一些资源
