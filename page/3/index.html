3.4 实时调度

  • 实时任务
    • 任务结束时间有严格约束,必须在deadline前完成
    • 具有紧迫性

  • 前述算法不能很好地满足实时系统对调度的特殊要求，所以引入实时调度

  • 实时操作系统 RTOS

    3.4.1 实现实时调度的基本条件

  1. 提供必要的调度信息

    1. 就绪时间 (该任务成为就绪状态的起始时间)
    2. 开始截止时间和完成截止时间
    3. 处理时间 (任务开始执行到完成所需时间)
    4. 资源要求
    5. 优先级 .若错过开始截止,则赋予绝对优先级

  2. 系统处理能力强

  3. 采用抢占式调度机制
    • 调度程序先调度开始截止时间即将到达的任务
  4. 具有快速切换机制
    1. 具有快速响应外部中断的能力
    2. 快速的任务分配能力

3.4.2 实施调度算法的分类

  1. 非抢占式调度算法
    1. 非抢占式轮转调度算法
      • ????????????????????//
      • 常用于要求不太严格的实时控制队列
    2. 非抢占式优先权调度算法
  1. 抢占式调度算法
    1. 基于时钟中断的抢占式优先权调度算法
    2. 立即抢占的优先权调度算法

3.4.3 常用的几种实时调度算法

最早截止时间优先(EDF)算法

  • 优先级确定 根据任务开始截至时间来确定任务的优先级,截止时间越早,优先级越高
  • 实时任务就绪队列 : 最早截止时间的任务排在队列最前
  • 调度顺序 : 总是选择就绪队列中的第一个任务,为之分配处理机,使之投入运行
  • 适用范围 : 即可适用于抢占式调度,也可适用于给抢占式调度
    1. 非抢占式调度方式用于周期实时任务

      最低松弛度优先(LLF)算法

  • 松弛度 = 完成截止时间-剩余运行时间-当前时间
  • 主要用于可抢占式调度方式

抢占方式和时机

  • 当有任务执行时,只有等待任务的松弛度为0时才会发生任务的调度,其他情况不发生调度
  • 任务执行结束后或无任务执行时,再比较等待任务的松弛度值,较小的先执行

3.3 调度算法

  • 调度算法 : 根据系统分配策略所规定的资源分配算法
  • 不同类型的系统和系统目标,采用不同的调度算法
  • 常用的调度算法
    1. 先来先服务
    2. 段作业(进程)优先
    3. 高优先-高响应比优先
    4. 时间片轮转
    5. 多级反馈队列(理解)
  • 调度算法有的适用于高级调度,有的适用于低级调度,有的既可用于高级调度,也可用于低级调度

3.3.1 先来先服务和短作业(进程)优先调度算法

  1. 先来先服务调度算法

    • 适用于作业调度和进程调度
    • 按照作业/进程进入系统的先后次序进行调度,先进入系统者优先;即启动等待时间最长的作业/进程
    • 优点
      • 有利于长作业
      • 有利于cpu繁忙型作业
    • 缺点
      • 不利于短作业.特别是来得较晚的短作业
      • 不利已I/o繁忙型作业
    • 用于批处理系统,不适于分时系统
    1. 短作业(进程)优先调度算法
    • 启动要求运行时间最短的算法
    • 分别用于作业调度和进程调度
    • 短作业优先(SJF)调度算法
      从后备队列中选择一个或若干个估计运行时间最短的作业,将它们调入内存运行
    • 短进程优先(SPF)调度算法
      从就绪队列中选择一估计运行时间最短的进程,将处理机分配给它,使它立即执行并一直执行到完成,或发生某事件而被阻塞放弃处理机时,再重新调度
    • 优点
      1. 能有效降低作业/进程的平均等待时间
      2. 提高系统的吞吐量
    • 缺点
      1. 对长作业不利,可能导致长作业长期不被调用
      2. 该算法完全未考虑作业的紧迫程度,因不能保证紧迫性作业(进程)会被及时处理
      3. 由于作业(进程)的长短只是根据用户所提供的估计执行时间而定的,而用户又会有意无意地缩短其作业的估计运行时间,导致该算法不一定能真正做到短作业优先调度
      4. 无法实现人机交互
  • 补充: 最短剩余时间优先调度算法(SRT)
    • 调度时选择预期剩余时间最短的进程,
    • 当一个新进场加入到就绪队列中时,他可能比当前正在运行的进程具有更短的剩余时间.因此,只要新进程就绪,调度器可能抢占当前正在运行的进程
    • 也可能存在长进程被饿死的危险

3.3.2 优先权调度算法PSA

  1. 优先权调度算法的类型
    1. 非抢占式优先权算法(用于批处理,要求不严的实时)
      • 系统一旦把处理机分配给就绪队列中优先权最高的进程后,该进程便一直进行下去,或因发生某事而使得该进程放弃处理机,系统方可再将处理机重新分配给另一优先权最高的进程
      • 主要用于批处理系统,也可用于某些对实时性要求不严的实时系统中
    2. 抢占式优先权算法(用于要求严格的实时,性能要求较高的批处理和分时)
      • 系统把处理机分配给就绪队列中优先权最高的进程,使之执行.但在其执行期间,只要出现了另一个优先权更高的进程,进程调度程序就立即停止当前进程(原优先权最高的进程)的执行,重新将处理机分配给新的优先权最高的进程.
      • 能更好的紧迫作业的要求
      • 只要系统中出现一个新的就绪进程,就会进行优先权比较
      • 用于要求严格的实时,性能要求较高的批处理和分时

  2. 优先权类型

    1. 静态优先权
      • 静态优先权是在进程创建时确定的,且在进程的整个运行期间保持不变
      • 确定进程优先权的依据
      1. 进程类型
        系统进程高,一般用户进程低
      2. 进程对资源的需求
        进程的估计执行时间,内存需求量等.要求少的进程赋予较高的优先权
      3. 用户要求
        紧迫程度,所付费用
      • 优点
      • 简单易行,系统开销小
      • 缺点
      • 不够精确,可能出现优先权低的作业或进程长期得不到调度的情况
    2. 动态优先权
      • 随进程的推进或其等待时间的增加而改变,以获得更好的调度性能
      • 可规定,在就绪队列中的进程,随着等待时间的增长,其优先权以某一速率提高
      • 相同优先权初值的进程进入就绪队列,FCFS算法
      • 不同优先权初值的就绪进程,则优先权低的进程,在等待了足够的时间后,其优先权便可提高到最高,从而获得处理机
      • 当采用抢占式优先权调度算法时,如果规定当前运行的进程的优先权以某一速率下降,则可防止一个长作业长期的垄断处理机

  3. 高响应比优先调度算法(HRRN High Response Ratio Next)(动态优先权机制)

    • 优先权的变化机制

      • 优先权 = (等待时间 +要求服务时间)/要求服务时间
      • 同时 优先权 = 响应比R_p
      • 所以 响应比 = (等待时间+ 要求服务时间)/要求服务时间 = 响应时间/要求服务时间

    • HRRN 是一种介于FCFS于SJP之间的算法,由于长作业也有机会投入运行,所以

      • 同一时间处理的作业数显然要小于SJF,
      • 从而采用HRRN方式时,吞吐量小于SJP,
      • 由于每次调度前要计算响应比,所以系统开销也要相应增加

    • 总结 :

      • 对短作业有利
      • 是先来先服务
      • 对长作业有利
      • 缺点 : 要进行计算响应比,增加系统开销

3.3.3 基于时间片的轮转调度算法

  1. 时间片轮转法
    1. 基本原理
      • 先来先服务
      • 分配时间片
      • 保证就绪队列中的所有进程，在一给定的时间内，均能获得一个时间片的处理机执行时间，换言之，系统能在给定的时间内，响应所有用户的请求。
    2. 时间片大小的确定
      • 考虑因素
      • 系统对响应时间的要求(用户数一定时,成正比)
      • 就绪队列中的进程数目(保证响应时间,成反比)
      • 系统处理能力(保证用户键入的命令能在一个时间片内处理完毕)
    3. 优缺点
      • 时间片大小对计算机性能的影响
      • 存在的问题 : 未有效利用系统资源,对于短的,计算型进程比较有利,因为该进程充分利用时间片,而I/O型进程却不利,因为在两次i/o之间仅需很少的cpu时间,却需要等待一个时间片
      • 常用于分时系统和事务处理系统

3.3.4 多级队列调度算法

  • 前台的就绪队列时交互性作业的进程,采用时间片轮转法
  • 后台的就绪队列是批处理作业的进程,采用优先权或短作业优先算法
  • 调度方式有两种
    • 优先调度前台,若前台无可运行进程,才调度后台
    • 分配占cpu的时间比例 ,如前台80%,后台20%

3.3.5 多级反馈队列调度算法

  • 调度算法
    1. 设置多个就绪队列,并为各个队列赋予不同的优先级.第一个队列的优先级最高,第二个次之,其余各队列的优先级逐个降低.规定在优先权越高的队列,每个进程的时间片就越小
    2. 一个新进程进入内存后,首先将它放入第一队列的末尾,按照FCFS原则等待调度.若该进程不能在该时间片内完成,则降入第二队列的末尾,以此类推,直到第n队列中就采用时间片轮转方式运行.
    3. 仅当第一队列空闲时,调度程序才调度第二队列中的进程运行,以此类推
  • 多级反馈队列调度算法过程
    1. 按优先级由高到底设置多个队列RQ_i,高优先级队列时间片小
    2. 刚进入系统的进程按FCFS放入最高的RQ_0中
    3. 进程一次时间片没执行完,就降至下一级队列,以此类推降至最低优先级队列后,不在下降
    4. 系统优先调度高优先级队列中的进程,仅当RQ_o中空闲时才调度RQ_1中的进程
  • 多级反馈队列调度算法的性能
    1. 终端型作业用户
      • 交互型作业用户,通常较小,第一队列一个时间片即可完成
      • 短批处理作业用户,第一队列一个时间片即可完成,或一二两个队列的时间片
      • 长批处理作业用户,

3.3.6 基于公平原则的调度算法

  1. 保证调度算法
    • 进程获得cpu的比率 = 进程已经执行的时间/(进程创建以来经历的时间/n)
    • 选择比率最小的,将处理机分配给他
  2. 公平分享调度算法
    • 以用户为单位进行调度

3.2 调度队列模型和调度准则

三种调度队列模型

  1. 仅有进程调度
  2. 具有高级和低级调度

  3. 具有三级调度

    3.2.1 调度队列模型

  4. 仅有进程调度的调度队列模型(分时系统)

    • 分时系统中,通常仅设有进程调度
    • 系统把这些进程组织成一个就绪队列
    • 每个进程在执行时,可能有以下几种情况
      • 进程获得cpu正在执行
      • 任务在给定的时间片内完成,释放处理机后为完成状态
      • 任务在给定时间片内未完成,进入就绪队列末尾
      • 在执行期间因某事件而阻塞
    • 就绪队列的时间片轮转
      • 常采用FCFS(FIFO)算法,FCFS(FIFO)队列
    • 进程执行时的三种情况
      • 完成,时间片到,阻塞

  5. 具有高级和低级调度的调度队列模型

    • 在批处理系统中,不仅需要进程调度,而且还要有作业调度
    • 就绪队列的形式
      • 在批处理系统中,常用高优先权队列.进程进入就绪队列,按优先权高低插入相应位置,调度程序总是把处理机分配给就绪队列首进程
    • 设置多个阻塞队列
      • 根据事件的不同设置多个阻塞队列提高效率
    • 常采用高优先权优先调度算法
      • 可采用优先队列,进程来时按优先权插队,从队首调度(效率高)
      • 可采用无序队列,每次调度时,先比较优先度
    • 多个阻塞队列

  6. 同时具有三级调度的调度队列模型

    • 在Os中引入中级调度后,进程的就绪状态分为内存就绪(表示进程在内存中就绪)和外存就绪(在外存中就绪)
    • 同时,阻塞状态分为内存阻塞和外存阻塞
    • 调出操作
      • 内存就绪-> 外存就绪
      • 内存阻塞-> 外存阻塞
    • 中级调度
      • 外存就绪 -> 内存就绪

3.2.2 选择调度方式和调度算法的若干准则

  • 在不同系统中通常采用不同的调度方式和算法
  • 调度的目标
    1. 提高处理机的利用率
    2. 提高系统的吞吐量
    3. 尽量减少进程的响应时间
    4. 防止进程长期得不到运行

  • 系统选择调度方式和算法的准则分为两种

    • 面向用户的准则
    • 面向系统的准则

  • 面向用户的准则

    1. 周转时间短(评价批处理)
      1. 作业在外存后备队列上等待调度的时间
      2. 进程在就绪队列等待调度的时间
      3. 进程在cpu执行的时间
      4. 等待i/o操作完成的时间
        • 带权周转时间 : 作业的周转时间T与系统为它提供服务的时间T_s之比,即W=T/T_s
        • 用来评价批处理系统的性能、选择作业调度方式与算法的重要准则之一

    2. 响应时间快(评价时分)

      1. 从键盘输入的请求信息传送到处理机的时间
      2. 处理机对请求信息进行处理的时间
      3. 将所形成的响应时间回送到终端显示器的时间
        • 响应时间 : 从用户通过键盘提交一个请求开始,知道系统首次产生响应所需要的时间
        • 用来评价分时系统的性能、选择进程调度算法的重要准则之一

    3. 截止时间的保证(评价实时)

      1. 开始截止时间
      2. 终止/完成截止时间
        • 截止时间 : 某任务必须开始执行的最迟时间,或必须完成的最迟时间.(deadline)
        • 用来评价实时系统的性能、选择实时调度算法的重要准则之一
    4. 优先权准则
      • 让某些紧急的作业能得到及时处理
      • 往往还需选择抢占调度方式,才能保证紧急作业得到及时处理
      • 适合批处理,分时,实时系统

  • 面向系统的准则

    1. 系统吞吐量高(评价批处理系统)
      • 吞吐量
      • 与批处理作业的平均长度有关
    2. 处理机利用率高.
      • 主要对大中型用户系统,对单用户和实时系统不重要
      • cpu利用率 = cpu有效工作时间/(cpu有效工作时间+ cpu空闲等待时间)
    3. 各类资源的平衡利用.(内存,外存,i/o设备等)
      • 主要对大中型系统,对微型机和实时系统不重要

  • 引起进程调度的因素可归结为

    1. 正在执行的进程执行完毕,或因发生某事件而不能再继续执行(包括:当前执行进程被中断,时间片用完了,挂起自己,推出等)
    2. 执行中的进程因提出i/o请求而暂停执行
    3. 在进程通信或同步过程中执行了某种原语操作,如p,v操作原语,block原语,wakeup原语等

  • 进程切换

    • 当一个进程占用处理机执行完(或不能继续执行),则换另一个进程占用处理机执行
    • 在进行时要保护进程的执行现场(进程的上下文)
  • 进程调度
    • 把处理机分配给不同的进程占用执行

  • 调度程序

    • 实现分配处理机的程序

  • 进程切换的基本步骤

    1. 保存进程上下文环境
    2. 更新当前运行进程的控制块内容,将其状态改为就绪或阻塞状态
    3. 将进程控制块移到相应队列(就绪队列或阻塞队列)
    4. 改变需投入运行进程的进程控制块内容,将其内容状态改变为运行状态
    5. 恢复需投入运行进程的上下文环境

  • linux的进程切换(context_switch())

    1. 进程页表PGD 切换
    2. 内核堆栈和硬件上下文切换(包含CPU寄存器)

3.1 处理机调度的层次

发表于 2018-05-08

Chapter3 处理机调度与死锁

3.1 处理机调度的层次

  • 在多道程序系统中,一个作业从提交到执行,通常都要经历多层调度,如
    • 高级调度 决定哪些程序可以进入系统
    • 中级调度 决定内存中程序的位置和状态
    • 低级调度 决定cpu内存中程序的位置和状态
    • i/o调度
  • 系统的运行性能在很大程度上取决于调度
    • 如吞吐量的大小,周转时间的长短,响应的及时性等
  • 调度是多道系统的关键

3.1.1 高级调度

  • 也称作业调度,长程调度或接纳调度

  • 主要功能

    • 根据某种算法,把外存中处于后备队列中的那些作业调入内存

  • 批处理系统需要有作业调度,分时和实时系统无需此调度

  • 每次执行作业调度时
    1. 接纳多少各作业(取决于多道程序度)
      • 多->服务质量下降
      • 少->资源利用率低
    2. 接纳哪些作业 (取决于采用的调度算法)

  • 名词:

    • 多道程序度 : 即允许多少个作业同时在内存中运行
    • 周转时间 : 从作业被提交给系统起,到作业完成为止的这段时间间隔
    • 吞吐量 : 是指在单位时间内系统所完成的作业数
    • 响应比 = (等待时间+要求服务时间)/要求服务时间 = 响应时间/要求服务时间

  • 主要用于批处理系统,其设计目标时最大限度地发挥各种资源地利用率和保持系统内各种活动地充分并行

3.1.2 低级调度

  • 又称进程调度或短程调度,它所调度地对象时进程,三种OS都必须配置这级调度(最基本调度)
  • 用于决定就绪队列中的哪个进程应获得处理机,然后再由分派进程执行把处理机分配给该进程的具体操作
  • 进程调度方式
    1. 非抢占方式 进程占用处理机直至自愿放弃或发生某事件被阻塞时,在把处理机分配给其他进程
    2. 抢占方式 允许暂停某个正在执行的进程,将处理机重新分配给另一个进程
  • 3个基本机制
    1. 排队器 为了提高进程调度的效率,应事先将系统中的所有就绪的进程按照一定的方式排成一个或多个队列
    2. 分派器(调度程序) 分派器把由进程调度程序所选定的进程从就绪队列中取出,然后进行上下文切换,将处理机分配给他
    3. 上下文切换机制 当对处理机进行切换时,会发生两对上下文切换操作
  • 功能
    • 按某种算法选区进程(调度)
    • 保存处理机的现场信息(上下文切换sept 1)
    • 把处理器分配给进程(上下文切换step 2)

  • 时间尺度 通常是毫秒级的
    -

  • 分配程序的主要功能

    • 进行进程切换
    • 转到用户态
    • 开始执行被选中的进程

  • 非抢占方式

    • 引起调度的因素
      • 正在执行的进程执行完毕,或者因发生某事件而不能再继续执行
      • 执行中的进程因提出i/o操作而暂停执行
      • 在进程通信或同步过程中执行了某种原语操作,如wait,block,wakeup原语
    • 优点
      • 算法简单,系统开销小
    • 缺点
      • 紧急任务不能及时响应
      • 短进程要等到长进程允许结束

  • 抢占方式

    • 优点
      • 可以防止一个长进程长时间占用处理机
      • 能为大多数进程提供更为公平的服务
      • 能满足对响应时间有着较严格要求的实时任务的需求
    • 缺点
      • 抢占方式比非抢占方式调度所需的开销较大
      • 调度算法复杂
    • 抢占的原则
      1. 时间片原则
        • 适用于分时,大多数实时以及要求较高的批处理系统
      2. 优先权原则
        • 重要紧急任务优先权高
      3. 短作业(进程)优先原则

3.1.3 中级调度

  • 又称中程调度
  • 主要目的
    • 提高内存的利用率和系统的吞吐量
  • 具体实现
    • 使那些暂时不能运行的进程不再占用宝贵的内存资源,而将其调至外存去等待,把此时的进程状态称为就绪驻外存状态或挂起状态.当这些进程又具备运行条件,且内存又稍有空闲时,由中级调度来决定把外存上的那些又具备运行条件的就绪进程,重新调入内存,并修改其状态为就绪状态,挂在就绪队列上等待进程调度

处理机调度的层次

  • 终端型作业：低级。
  • 批量性作业：高级—低级。
  • 现代较完善的os具有三级调度。
